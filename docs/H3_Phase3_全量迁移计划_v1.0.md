## 1. 第一阶段: 现有 165M 重推理

### 1.1 推理资源估算

| 项 | 值 |
|----|----|
| 数据量 | 165M 图 |
| 单 GPU 速率 | 520 imgs/s (DINOv2-L fp16, 4090D) |
| 4 GPU 并行 | ~2000 imgs/s |
| **预计耗时** | **165M / 2000 = 82,500 秒 ≈ 23 小时** |

### 1.2 推理流程

```
1. 导出现有 image_pk + url 列表 (从 PG uri_dedup 表)
   预计 ~165M 行, ~12 GB

2. 按 Collection 分批 (按月份/类别)
   - 每批 ~10M-50M 图
   
3. 4 GPU 并行下载 + 推理
   - 复用 scripts/run_g2_g10_fast.py 框架
   - 但下载是从图床拉, 不是本地文件
   - 限制并发避免被限流
   
4. 输出 jsonl: {image_pk, dinov2_vec_1024, ts_month, ...}

5. 写入新 Collection (parallel to 旧 CLIP collection)
```

### 1.3 关键挑战

| 挑战 | 解决方案 |
|------|---------|
| 图片重新下载 165M | CDN 缓存命中, 大部分快; 失败的需要重试 |
| 长时间运行稳定性 | 断点续传 (按 image_pk 已处理状态) |
| 双写期间一致性 | 旧 Collection 仍在线, 新 Collection 静默写 |
| Milvus 写入吞吐 | 单 Collection 写入 ~2000/s, 总 165M 需 ~23h |

### 1.4 节省方案

**复用已有图片避免重新下载**:
```
现有图片:
  /data/imgsrch/dl_batch1 (988K, 已删除)
  /data/test-images/ (924K)
  /tmp/import_g{6-10}/ (4.68M, 已删除)

可重用: ~5.6M (非常少, 大部分已清理)
```

**结论**: 几乎都需要重新下载, 总流量 ~165M × 50KB = ~8 TB
- 内网下载假设 100 MB/s = 22 小时
- 外网下载假设 5 MB/s = 18 天 (不可行)

**必须**: 用 COS 内网直连 (假设有), 或本地缓存图片到 SSD 后批量推理

## 2. 第二阶段: 扩展到 30 亿

### 2.1 数据来源
- 现有 165M (新推理后)
- 历史归档 ~5亿 (需要从备份恢复)
- 增量数据 ~25亿 (新平台/品类拓展)

### 2.2 推理资源 (按 5 机集群)

| 项 | 单机 | 5 机集群 |
|----|------|---------|
| GPU | 4 × 4090D | 20 × 4090D |
| 推理速率 | 2000 img/s | **10,000 img/s** |
| 30 亿耗时 | 415 小时 | **83 小时 ≈ 3.5 天** |

### 2.3 数据传输

| 数据流 | 量 | 方案 |
|-------|---|------|
| COS → 5 机 | 8 TB | 内网直连, 每机 1.6 TB 入流量 |
| 推理后 vector → Milvus | 30B × 1.3 KB = 4 TB | 跨机分片写入 |
| Milvus 索引构建 | 4 TB → 内存 + SSD | 自动 |

## 3. 上线灰度方案

### 3.1 影子流量阶段 (1 周)

```
旧服务 (CLIP):  100% 用户流量, 100% 返回结果
新服务 (DINOv2): 复制旧服务 100% 流量, 静默执行不返回

对比指标:
  - Recall@10 (新 vs 旧)
  - 延迟 (P50/P95/P99)
  - 错误率
```

### 3.2 灰度 1% (1 周)

```
路由器:  99% 旧服务 + 1% 新服务
监控:    Recall, 延迟, 用户反馈
回滚条件: Recall < 95% 或 P95 > 100ms
```

### 3.3 灰度 50% (3 天)

```
路由器:  50/50 AB 测试
对比:   两组用户的点击率/转化率
```

### 3.4 全量切换

```
新服务 100%, 旧服务降级为备份
观察 1 周后下线旧 Collection
```

## 5. 风险与缓解

| 风险 | 影响 | 缓解 |
|------|------|------|
| CDN 限流 | 推理慢/失败 | 按 IP 分散下载, 限速 |
| Milvus 写入 OOM | 服务中断 | 监控 RAM, 调小 batch |
| GPU 故障 | 推理中断 | 4 GPU 高可用, 单卡故障切换 |
| 索引构建慢 | 上线延迟 | 预先构建 + 离线导入 |
| 灰度回滚 | 用户体验抖动 | 自动回滚阈值 |
| 双写一致性 | 数据不一致 | 双写脚本验证 + 校验工具 |
