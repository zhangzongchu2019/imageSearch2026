# 方案 H4 — 极简版向量检索架构 v0.3

- **创建日期**: 2026-04-10
- **版本**: v0.3 (综合 GPT + Gemini + Kimi 三家评审修订)
- **状态**: 验证中
- **定位**: 面向实例近重复搜索的极简生产候选方案

---

## 0. 核心理念

**两个模型, 搞定一切。**

| 模型 | 用途 | 速度 |
|------|------|------|
| **SSCD ResNet50** | 向量 (搜索排序) | 500/s per GPU |
| **CLIP ViT-L-14** | 标签 (语义过滤) | 350/s per GPU |

不存 CLIP/DINOv2/SigLIP 向量, 不用 VLM, 不用 Grounding DINO。

### 能力定位 (明确边界)

**核心能力 (承诺)**:
- ✅ 原图 / 压缩 / 裁剪 / 轻形变 / 涂鸦 / 水印 → R@10 99%+
- ✅ 重度裁剪 30% → R@10 99%+ (SSCD 强项)
- ✅ 粗语义过滤后实例排序 (品类+颜色+SSCD)
- ✅ 精确去重 (pHash)

**增强能力 (非默认, 需额外配置)**:
- ⚠️ 图中图 → 中心裁剪辅助 (30-50%), 需 L2/L3 增强
- ⚠️ 反图中图 → SSCD 57%, 需子图向量增强

**非目标 (不承诺)**:
- ❌ 强语义跨款搜索 ("适合海边的裙子")
- ❌ 细粒度属性对比 ("类似但领子不同")
- ❌ 高精度材质/风格判断 (CLIP 70-80%)

## 1. 架构总览

```
每张图入库 (实时, 120s SLA):
  SSCD 全图        → sscd_vec (512d)     → Milvus
  SSCD 中心60%     → center_vec (512d)   → Milvus (图中图辅助, L1)
  CLIP 零样本分类  → 强标签 + 弱标签     → Milvus
  pHash + dHash    → 128bit 指纹        → Milvus
  quality/scene    → 质量分 + 场景类型   → Milvus
  
搜索 (80 QPS, 终端用户直接搜索):
  mode=instance           → SSCD 向量搜索
  mode=filtered_instance  → 强标签过滤 + 弱标签 boost + SSCD 排序
  mode=exact              → pHash 汉明距离 + SSCD 兜底
  mode=auto               → 强标签过滤 + SSCD 排序 (默认)
```

## 2. 数据模型

### 2.1 向量数据

| 字段 | 类型 | 大小 | 来源 | 用途 |
|------|------|------|------|------|
| sscd_vec | FLOAT_VECTOR 512d SQ8 | 512 B | SSCD ResNet50 | 实例/副本搜索排序 |
| center_vec | FLOAT_VECTOR 512d SQ8 | 512 B | SSCD (中心裁剪60%) | 图中图辅助 |
| phash | BINARY_VECTOR 64bit | 8 B | CPU 计算 | 原图极速去重 |
| dhash | BINARY_VECTOR 64bit | 8 B | CPU 计算 | 原图极速去重 |

### 2.2 强过滤标签 (CLIP, 准确率 >90%, 用于 WHERE)

| 字段 | 类型 | 大小 | 说明 |
|------|------|------|------|
| category_l1 | INT32 | 4 B | 服装/箱包/鞋帽/珠宝/家具/灯具/化妆品 (~95%) |
| category_l2 | INT32 | 4 B | T恤/连衣裙/手提包/运动鞋... (~85%) |
| color_code | INT32 | 4 B | 红/蓝/黑/白/多色... (~90%) |

### 2.3 弱过滤标签 (CLIP, 准确率 60-85%, 仅 score boost)

| 字段 | 类型 | 大小 | 说明 | 准确率 |
|------|------|------|------|--------|
| material_code | INT32 | 4 B | 棉/涤纶/皮革/金属/木 | ~70% |
| pattern_code | INT32 | 4 B | 纯色/条纹/格子/印花/碎花 | ~80% |
| style_code | INT32 | 4 B | 休闲/商务/运动/复古 | ~75% |
| sleeve_code | INT32 | 4 B | 短袖/长袖/无袖 (仅服装) | ~80% |
| season_code | INT32 | 4 B | 春夏/秋冬/四季 | ~60% |
| gender_code | INT32 | 4 B | 男/女/中性 | ~80% |

**标签置信度规则**: CLIP 分类概率 <0.7 时标记为 0 (未知), 不参与过滤/打分。

### 2.4 辅助字段 (v0.3 新增)

| 字段 | 类型 | 大小 | 说明 |
|------|------|------|------|
| scene_type | INT32 | 4 B | 0=未知 1=白底 2=模特 3=场景 4=截图 5=拼图 |
| quality_score | INT32 | 4 B | 0-100 分 (清晰度+信息量) |

### 2.5 业务元数据

| 字段 | 类型 | 大小 | 说明 |
|------|------|------|------|
| image_pk | VARCHAR(64) | ~32 B | 图片唯一 ID (md5) |
| url_hash | VARCHAR(64) | ~32 B | URL 哈希 |
| brand_id | INT32 | 4 B | 品牌 ID |
| merchant_set_id | INT64 | 8 B | 商家集合哈希 (外挂 bitmap 关联) |
| ts_month | INT32 | 4 B | 时间分区 |
| created_at | INT64 | 8 B | 创建时间 |

### 2.6 商家过滤 (外挂层, 不在主表)

```
主表: 只存 merchant_set_id (INT64)

外挂层 (Redis / 独立 Collection):
  merchant_id → RoaringBitmap(image_pk 集合)

查询: 先从外挂取 image_pk 集合 → 传给 Milvus 做 bitset 过滤

原因: BITMAP 索引不适合高基数字段 (Milvus 建议 <500 distinct values)
```

### 2.7 存储汇总

```
单图数据:                   ~1,172 B
索引开销 (M=24, 双向量):      ~464 B
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
单图总计:                   ~1,636 B

工程预算区间: 1.6 KB ~ 2.0 KB
(上线前必须用 1000 万级真实数据回归实测)
```

## 3. 索引配置 (v0.3 修订)

```python
# 主向量 SSCD (M=24, v0.1 为 48, v0.3 降低省内存)
sscd_index = {
    "index_type": "HNSW_SQ",
    "metric_type": "COSINE",
    "params": {
        "M": 24,                # v0.1:48 → v0.3:24 (省 1.15TB/30亿)
        "efConstruction": 200,  # v0.1:1000 → v0.3:200
        "sq_type": "SQ8",
        "refine": True,         # 弥补 M 降低的精度
        "refine_type": "FP16",
    }
}
center_index = sscd_index  # 同配置

phash_index = {"index_type": "BIN_FLAT", "metric_type": "HAMMING"}
brand_index = {"index_type": "STL_SORT"}
category_index = {"index_type": "STL_SORT"}
color_index = {"index_type": "STL_SORT"}

search_params = {"ef": 200}
```

## 4. 搜索策略

### 4.1 mode=instance (实例搜索)

```
用途: 找原图/裁剪/截图/同款/视角变体
流程: query → SSCD encode → 搜索 sscd_vec + center_vec → 合并 Top K
延迟: ~20ms 端到端 (含网络+编码+检索+序列化)
```

### 4.2 mode=filtered_instance (过滤后实例排序, 原 semantic)

```
用途: 找"类似的红色T恤" / 同品类同颜色
流程: 
  强过滤: WHERE category_l2 = T恤 AND color_code = 红
  弱打分: boost += 0.1 × (material 匹配 + pattern 匹配 + ...)
  排序:   final = 0.8 × SSCD_score + 0.2 × boost_score

标签 fallback: 若过滤后结果 <5 条, 去掉标签做全库 SSCD 扫描
延迟: ~25ms (正常), ~50ms (fallback)
```

### 4.3 mode=exact (精确搜索)

```
用途: 找完全相同的图 (原图, 压缩, 水印)
流程: query → pHash → 汉明距离 < 5 → 如无命中 fallback 到 SSCD
延迟: ~5ms
```

### 4.4 mode=auto (智能融合, 默认)

```
流程: 强标签过滤 (如果查询图能提取标签) + SSCD 排序
等效于 filtered_instance, 但标签提取失败时降级为 instance
```

| 场景 | 验收标准 | SSCD R@K | 达标? |
|------|---------|---------|------|
| 原图 | Top 1 = 原图 | R@1 96.9% + pHash 100% | ✅ |
| 压缩/转发 | Top 3 | R@3 99.5% | ✅ |
| 轻度裁剪 70% | Top 8 | R@8 99.8% | ✅ |
| 轻度形变 | Top 10 | R@10 100% | ✅ |
| UI叠加/涂鸦 ≤30% | Top 5 | R@5 99.7% | ✅ |
| 重度裁剪 30% | Top 10 | R@10 99.3% | ✅ |
| 重度涂鸦 | Top 5 | R@5 99.2% | ✅ |
| 分辨率缩放 | Top 10 | R@10 100% | ✅ |

### 图中图增强路径

```
L1 (默认): 全图 SSCD + center_vec (30-50%)  ← 不保证达标
L2 (可选): 异步 YOLO 检测 + 子图向量 (60-80%)
L3 (远期): DINOv2 Patch 级匹配 (70-90%)
```

## 9. 数据生命周期 (v0.3 新增)

```
商品下架:
  1. 业务消息 → Milvus delete(image_pk)  (软删除)
  2. 外挂 merchant bitmap 同步更新
  3. 定期 compact (每周): 物理回收碎片

图片替换:
  1. delete 旧向量 + insert 新向量 (image_pk 不变)
  2. SSCD + CLIP 重新推理

定期维护:
  - 每周: Milvus compact
  - 每月: 索引重建 (可选, 优化 HNSW)
  - 监控: 碎片率 > 30% 触发告警
```

## 12. 风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| CLIP 标签 85% 不足 | 语义搜索漏 15% | fallback 全库扫 / VLM 纠错 |
| 图中图 30-50% 不够 | 截图搜索弱 | 异步 YOLO 子图 (L2) |
| SSCD 非服装品类退化 | 家具/灯具召回低 | 跨品类评测 + DINOv2 冷备 |
| HNSW M=24 精度下降 | R@10 降 ~1% | refine=FP16 弥补 |
| 存储超预算 1.6→2.0KB | 单机 RAM 超 37% | 实测后调整 M 或维度 |
| Milvus 碎片化 | 性能退化 | 每周 compact + 监控 |
