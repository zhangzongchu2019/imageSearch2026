# 全链路测试设计 v1.0

> 版本: v1.0 | 日期: 2026-04-04 | 系统: 以图搜商品 v0.3

## 1. 测试目标

在接近生产规模 (4:1 缩比) 的环境下验证:
- 数据写入链路完整性 (上传 → 推理 → 入库 → 可搜索)
- 搜索链路准确性 (查询 → 向量匹配 → 返回相似图)
- 系统在持续写入和查询并发下的稳定性
- 分区隔离正确性 (VIP / SVIP / 常青 / 冷区)

## 4. 衍生数据规则

### 4.1 衍生目的

模拟同品类、不同商户的相似商品。每条真实记录衍生 N 个变体, 使总量达到目标。

### 4.2 衍生参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 目标总量 | 2000 万/分区 | 含真实记录 |
| 真实记录数 | 40~50 万/分区 | GPU 推理得到 |
| 衍生倍率 | ~39~49× | 自动计算: (目标 - 真实数) / 真实数 |
| 随机种子 | 2026 | 确定性 (相同输入 → 相同输出) |
| 分批大小 | 100 万/批 | 避免 cgroup 64GB OOM |

### 4.3 向量衍生

```python
# 对每条真实记录的 256 维向量:
noise = np.random.normal(0, σ=0.03, size=256)   # 高斯噪声
perturbed = original_vec + noise                  # 加噪声
perturbed = perturbed / np.linalg.norm(perturbed) # L2 归一化

# σ=0.03 的含义:
#   cosine_similarity(original, perturbed) ≈ 0.9985
#   搜索时 perturbed 会排在 original 附近但不完全重叠
```

### 4.4 品类衍生

```python
# L1 品类: 100% 继承源记录 (不改变品类)
v_category_l1 = source.category_l1

# L2/L3: 当前为 0 (FashionSigLIP 批量导入时跳过)
v_category_l2 = 0
v_category_l3 = 0
```

### 4.5 标签衍生

```python
# 80% 概率: 完全继承源记录标签
v_tags = source.tags.copy()

# 20% 概率: 微调 1 个标签
if random() < 0.2:
    if tags and random() < 0.5:
        tags.pop(random_index)        # 随机删除 1 个
    else:
        tags.append(random_tag_id)    # 随机添加 1 个 (范围 100-4095)
    tags = tags[:32]                  # 最多 32 个
```

### 4.6 商户衍生

```python
# 真实记录: merchant_id 范围 10001-12999 (2999 个商户)
# 衍生记录: merchant_id 范围 10001-15999 (5999 个商户, 更大范围)
merchant_num = random.randint(10001, 15999)
merchant_str = f"T20260101{merchant_num}"
```

### 4.7 主键衍生

```python
# 变体 PK = SHA256(原始PK + "_v{全局序号}") 的前 16 字节 hex
vpk = hashlib.sha256(f"{original_pk}_v{global_idx}".encode()).digest()[:16].hex()
# 保证全局唯一, 且可溯源到源记录
```

### 4.8 衍生数据 vs 真实数据对比

| 维度 | 真实记录 | 衍生记录 |
|------|---------|---------|
| 向量 | GPU 真实提取 (768→256) | 原始 + 高斯噪声 (σ=0.03) |
| L1 品类 | GPU 零样本分类 (18 类) | 100% 继承 |
| L2/L3 | 0 (FashionSigLIP 跳过) | 0 |
| 标签 | GPU 零样本提取 (top-8) | 80% 继承 / 20% 微调 1 个 |
| 商户 | 随机 10001-12999 | 随机 10001-15999 |
| URI | 真实 COS URL | synthetic://原PK/v序号 |
| is_evergreen | false | false |
| ts_month | 202604 | 与分区一致 |

## 5. 全链路测试计划

### 5.1 测试一: 全链路接口联调 (Task 10)

**目标**: 验证写入+查询走正常服务接口, 不直连 Milvus

**写入测试**:
```
图片文件 → POST /api/v1/images (write-service)
         → inference-service 推理 (ViT-L-14)
         → write-service 写 Milvus + PG
         → 验证 5-10 秒后可搜索
```

**查询测试**:
```
图片文件 → POST /api/v1/search (search-service)
         → inference-service 提取查询向量
         → Milvus ANN + bitmap 过滤
         → 返回 top-K 相似图
```

**验证项**:
- [ ] 写入后 5-10 秒内可搜索
- [ ] 搜索结果包含写入的图片
- [ ] VIP/SVIP 分区隔离 (指定 partition 搜索)
- [ ] 品类过滤正确 (category_l1 filter)
- [ ] 标签过滤正确 (tags filter)

### 5.2 测试二: 批量写入 1000 万 (Task 11)

**目标**: 模拟生产环境日常批量更新

**数据规划**:
| 分区 | 真实图片 | 衍生倍率 | 总量 |
|------|---------|---------|------|
| 普通 (p_202604) | 25 万 | ×20 | 500 万 |
| SVIP (p_202604_svip) | 25 万 | ×20 | 500 万 |
| **合计** | 50 万 | | 1000 万 |

**URL 来源**: urls_cos.txt 中未使用的部分 (offset 900000+)

**流程**:
1. 提取 50 万 URL (25 万普通 + 25 万 SVIP)
2. 通过 write-service API 批量写入 (不直连 Milvus)
3. 每批 1000 条, 控制写入速率 (≤200 QPS)
4. 验证: 写入后搜索可达, 数据量增长正确

**验证项**:
- [ ] 写入速率稳定 (无超时/失败)
- [ ] Milvus/PG 数据量增长正确
- [ ] 写入期间搜索延迟无明显上升
- [ ] Milvus etcd 无超时告警

### 5.3 测试三: 查询测试 50000 条 (Task 12)

**目标**: 大规模验证搜索准确性和原图匹配率

**测试集生成**:
```python
# 从 1000 万 URL 中每隔 200 条取 1 条
# 总共 50,000 条测试 URL
test_urls = urls_cos[::200][:50000]

# 其中:
#   ~2000 条 URL 在 VIP 分区有原图 (offset 0-400000, 间隔 200)
#   ~2500 条 URL 在 SVIP 分区有原图 (offset 400000-900000, 间隔 200)
#   ~45500 条 URL 不在任何分区 (应该搜到相似图但非原图)
```

**测试流程**:
```
对每条测试 URL:
  1. 下载图片 (COS 缩图 224px)
  2. POST /api/v1/search → 获取 top-10 结果
  3. 判断: top-1 结果是否为原图 (cosine similarity > 0.99)
  4. 记录: 命中/未命中, 延迟, top-1 品类
```

**预期结果**:
| 场景 | 数量 | 预期 top-1 行为 |
|------|------|----------------|
| URL 在库中有原图 | ~4500 | cosine > 0.99, **应该命中** |
| URL 不在库中 | ~45500 | cosine < 0.99, 返回相似图 |

**验证指标**:
- [ ] 原图命中率 > 95% (有原图的 URL)
- [ ] 搜索延迟: P50 < 10ms, P95 < 50ms, P99 < 100ms
- [ ] 品类预测准确率 (top-1 品类 vs 源图品类)
- [ ] 无搜索超时/报错
