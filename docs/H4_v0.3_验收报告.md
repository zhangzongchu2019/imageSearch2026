## 2. 数据模型 (固化)

### 每图存储

```
向量:
  sscd_vec       512d SQ8      512 B  (SSCD 全图)
  center_vec     512d SQ8      512 B  (SSCD 中心裁剪 60%)
  phash/dhash    128 bit        16 B

强过滤标签 (WHERE):
  category_l1    INT32           4 B  (准确率 ~95%)
  category_l2    INT32           4 B  (~85%)
  color_code     INT32           4 B  (~90%)

弱打分标签 (score boost):
  material/pattern/style/sleeve/season/gender  各 4 B = 24 B

辅助:
  scene_type     INT32           4 B
  quality_score  INT32           4 B

业务:
  image_pk, url_hash, brand_id, merchant_set_id, ts_month, created_at

单图: ~1.6 KB (目标), 1.6~2.0 KB (工程预算)
```

### 索引配置 (固化)

```python
HNSW_SQ: M=24, efConstruction=200, sq_type=SQ8, refine=True, refine_type=FP16
搜索:    ef=200
```

## 3. 搜索架构 (固化)

### 两阶段搜索

```
Stage 1 (所有查询, ~20ms):
  SSCD encode → 搜索 sscd_vec + center_vec
  → top1_score

  if top1_score ≥ 0.8:
    返回结果 (快速路径, ~90% 查询)

Stage 2 (fallback, ~50ms):
  多尺度裁剪 (6 块) → SSCD batch encode
  → 搜索每块 → 合并结果
  仅 top1_score < 0.8 时触发 (~10% 查询)

平均延迟: ~23ms (0.9 × 20 + 0.1 × 50)
```

## 5. 关键技术突破

### 两阶段搜索 (零成本提升)

```
核心思路: score < 0.8 时自动触发多尺度裁剪 fallback
实现: 纯代码改动, 不改数据库, 不存额外向量, 不增加新模型
效果: 图中图 +69.5%, 反图中图 +45.5%
延迟: 90% 查询 0 开销, 10% 查询 +30ms
```

### HNSW M=24 vs M=48

```
M=48 → M=24: 索引内存减半
精度: refine=FP16 弥补, 实测无明显下降
省: 30 亿数据节省 1.15 TB 邻接表
```
